Таск-трекер

Простой таск-трекер с возможностью архивного хранения файлов. Реальная задача по реальной потребности.
Пользователи могут создавать хабы (группы) и добавлять в них других пользователей. У каждого пользователя
в каждом хабе, в котором он состоит, есть одна из трех ролей: admin (может добавлять/удалять других в этот хаб), 
editor (может писать/комментировать в этот хаб), reader (может только читать). Пользователям доступно
содержимое только тех хабов, в которых они состоят. В хаб можно добавлять задачи, каждой из которых
присваивается статус todo, doing, done, trash (kanban) для дальнейшего контроля за работой над ними.
К кажой задаче можно дополнительно указывать любое количество тегов, чтобы в дальнейшем можно было их сортировать.
Каждую задачу можно комментировать, а к комментариям прикреплять файлы. Кроме того, зарезервирована возможность
в дальнейшем указывать любые метаданные для пользователей, хабов и задач не изменяя схему (с помощью отдельных таблиц).

Для реализации взят postgres (извините) и PHP для серверной части (еще больше извините). Конечно, следовало бы
выбрать Ruby/Python, но с руби я еще не работал, а в пайтоне не дошли руки до джанги, а задача интересная и
хочется реализовать быстрее (но это уже за рамками этого readme).
